## About

This project demonstrates quantum computing concepts using [Qiskit](https://www.ibm.com/quantum/qiskit), comparing classical and quantum approaches to determining whether a Boolean function is *constant* or *balanced*. The main deliverable is a Jupyter notebook ([problems.ipynb](problems.ipynb)) that builds up from generating random Boolean functions in plain Python, to constructing quantum oracles, to running the full Deutsch–Jozsa algorithm on a four-bit input.

## Problems Covered

1. **Generating Random Boolean Functions** — `random_constant` and `random_balanced` generators for four-bit functions, plus a combined `random_constant_balanced` selector.
2. **Classical Testing for Function Type** — Determining whether a function is constant or balanced using classical queries, and analysing the worst-case query count.
3. **Quantum Oracles** — Implementing quantum oracles $U_f$ for single-bit Boolean functions $f: \{0,1\} \rightarrow \{0,1\}$ in Qiskit.
4. **Deutsch's Algorithm with Qiskit** — Building and simulating the quantum circuit that solves Deutsch's problem with a single oracle query.
5. **Scaling to the Deutsch–Jozsa Algorithm** — Generalising the construction to four-bit Boolean functions and verifying correctness against the classical results from Problem 2.

## Setup Instructions

### Prerequisites

- Python 3.10 or higher
- `pip` package manager
- (Optional) [Jupyter](https://jupyter.org/) or VS Code with the Jupyter extension

### Installation

1. Clone this repository:
   ```bash
   git clone https://github.com/CianDickerHughes/EMERGING-TECHNOLOGIES-2026.git
   cd EMERGING-TECHNOLOGIES-2026
   ```

2. (Recommended) Create and activate a virtual environment:

   **macOS / Linux:**
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   ```

   **Windows (cmd):**
   ```bat
   python -m venv .venv
   .venv\Scripts\activate.bat
   ```

3. Install required packages:
   ```bash
   pip install -r requirements.txt
   ```

4. Launch Jupyter Notebook (use `python -m` so it works even if `jupyter` is not on `PATH`):
   ```bash
   python -m jupyter notebook problems.ipynb
   ```

   Alternatively, open [problems.ipynb](problems.ipynb) directly in VS Code and select the `.venv` interpreter as the kernel.

Run the cells in order from top to bottom. Generated circuit diagrams and plots are written to the [img/](img/) directory.
